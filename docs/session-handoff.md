# セッション引き継ぎ / ログアウト・ポリシー

> Claude Code のアカウント切替・マシン変更・Codex への交代・日をまたぐ中断——
> すべて同じ1つの手順で安全にできる。**このプロジェクトの記憶は git と md にあり、
> アカウントには無い**、が大原則。

## 何が引き継がれ、何が引き継がれないか

| もの | 引き継ぎ | 備考 |
|---|---|---|
| コード・戦略文書・進捗ログ（CLAUDE.md / PROGRESS.md / TODO.md 等） | ✅ git + GitHub | push 済みであれば誰でも・どこでも |
| プロジェクト設定（hooks・devcontainer・Makefile） | ✅ リポジトリ内 | `make setup` で再有効化 |
| Claude Code のローカルメモリ・会話履歴 | △ 同じMacなら残る | `~/.claude` 配下。別マシンには行かない（だから md に書く運用にしている） |
| アカウント側のもの（利用枠・プラン） | ❌ | 切替の影響はこれだけ |

## ログアウト前の手順（毎回これだけ）

```bash
make handoff
```

**handoff とは**: `scripts/handoff-check.sh` による一括点検。中身は4つ——
1. **競合コピー検出**（iCloud が作る `xxx 2.js` 等。あれば diff 確認のうえ削除）
2. **未コミットの変更**が無いか
3. **未 push コミット**が無いか（push しないと他所から見えない）
4. **`npm run check`**（テスト＋週次JSON検証＋lint＋整形 = CI 相当）

結果の読み方: `✅ 引き継ぎ準備OK` → そのままログアウトしてよい。`⚠️` → commit/push してから。`❌` → テストを直してから（この状態でのログアウトは禁止）。

なお **Stop フック**（`.claude/settings.json` → `handoff-hook.sh`）が Claude Code の停止のたびに同種の警告を自動表示するので、忘れても気づける。

## アカウント切替の手順

1. `make handoff` → ✅ を確認
2. 同じターミナルで `/logout` → `/login`（**別ウィンドウは不要**。ログイン状態はマシン単位で全ウィンドウ共通のため、「別ウィンドウなら安全」ということはない）
3. 新セッションの最初の一言は「CLAUDE.md を読んで現状を把握して」だけでよい（CLAUDE.md は自動読込されるが、明示すると確実）

## 新しいセッション（誰でも）の立ち上がり方

読み順: **CLAUDE.md**（制約・進捗・次のタスク）→ **PROGRESS.md**（直近スプリントの詳細）→ **TODO.md**（人間の残作業）。エージェント向け入口は AGENTS.md、コードの地図は docs/DEVELOPMENT.md。

## 禁止事項

- ❌ 未 push のままログアウト／マシン変更（作業が見えなくなる）
- ❌ `npm run check` が赤いままセッションを終える（次の人が原因を切り分けられない）
- ❌ 競合コピー（` 2.` 付きファイル）を放置（後で正本と混ざる。恒久対策はリポジトリの iCloud 外移設＝TODO ☐7）
