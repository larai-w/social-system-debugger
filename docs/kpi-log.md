## イベントが飛ぶ仕組み（現状）

- アプリ内の `track(event, props)` は **`window.plausible` が存在するときだけ**イベントを送る（無ければ完全 no-op）。
- 計測イベント一覧は README の計測表を参照（weekly_start / weekly_clear / weekly_fail / share_x / share_line / card_saved / share_other / export_json / export_csv / notification_optin など。全てに `app_platform` 付き）。
- つまり**アナリティクスのスクリプトを有効化するまで、この表のイベント列は埋まらない**（訪問数すら取れない）。有効化はユーザー作業（TODO ☐9）。
