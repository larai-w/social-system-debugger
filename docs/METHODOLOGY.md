# 方法論・モデル仕様 / Methodology

> **[日本語](METHODOLOGY.md)**（このページ）｜**[English](METHODOLOGY.en.md)**
>
> このアプリ「社会＆地域インフラ・デバッガー」で使われている**すべての計算式・しきい値・前提**を一箇所にまとめた技術ドキュメントです。ソース（`index.html`）と同じ内容を、読みやすい形で明示します。

---

## ⚠ この文書の最重要ポイント（先に読んでください）

このモデルは **定性的な"思考実験"** です。次を明確にしておきます。

- **実測較正はしていません。** 各係数（例: `f*42`、`sf*52`）は、現象の方向性を分かりやすく見せるために**手で調整した説明用の値**であり、統計的に推定したパラメータではありません。
- **予測ツールではありません。** 出力される数値（エントロピー87、インフラ12 など）に**絶対的な意味はなく**、「この設計を強めると、この指標はこの向きに動く」という**関係性の可視化**が目的です。
- **因果を主張しません。** 実社会の因果を証明するものではなく、システム工学・機械学習の概念を社会現象に**メタファーとして写像**したものです。
- **特定の誰か・特定の政治的立場の話ではありません。** 「どの社会にも起こり得る構造」を扱う教育教材です（アプリ内の共有文にも同趣旨の注記があります）。

**モデルの前提を批判的に検討すること自体が、この教材の学習課題です。**

---

## 全体設計

社会を **1つの分散システム（OS）** に見立て、4つのレイヤーで構成します。各レイヤーは「ユーザーが動かすパラメータ → 計算式 → メトリクス → 判定（危機/注意/模範）」という流れを持ちます。

| Layer | 名称 | 中心テーマ | 主なメタファー |
|---|---|---|---|
| **L1** | 情報空間 | エコーチェンバー／過学習 | Overfitting・モード崩壊 |
| **L2** | 物理インフラ | 効率化と冗長性・他責デッドロック | Redundancy・Deadlock |
| **L3** | 個人の認知 | 毒入れ攻撃・思考の更新停止 | 探索深度・学習率(勾配) |
| **L4** | 当事者性 | 部外者ノイズによる声の埋没 | Sybil攻撃・目的関数の乗っ取り |

`clamp(x, 0, 100)` は「0〜100に丸める」関数です（以下すべてで使用）。

---

## L1 — 情報空間（`metrics(fr, es, al)`）

**パラメータ**
- `f = filterRate/100` … 情報フィルタリング率（エコーチェンバー度）
- `e = ethicsScore/100` … リーダーの倫理観（1=法治、0=ポピュリズム）
- `isG` … 意思決定アルゴリズムが Greedy か（true/false）
- `gp = isG ? 0.38 : 0` … Greedy ペナルティ係数

**主要メトリクス（抜粋・実コードと一致）**

```
entropy    = clamp(f*42 + (1-e)*38 + gp*20)      // 無秩序度
paranoia   = clamp(f*48 + (1-e)*33 + gp*14)
socialCap  = clamp(100 - f*30 - (1-e)*42 - gp*28) // ソーシャルキャピタル
infra      = clamp(100 - (1-e)*52 - f*20 - gp*28)
diversity  = clamp(100 - f*82)                    // 情報多様性
infoH      = clamp((1-f)*90 + e*10)               // 情報健全性
trust      = clamp(e*80 + (1-f)*20)
resilience = clamp(e*52 + (1-f)*48 - gp*30)
legitimacy = clamp(e*92 + (1-f)*8)
viability  = clamp(e*62 + (1-f)*38 - gp*42)
mentalWB   = clamp(100 - paranoia*0.92)
dopamine   = isG ? clamp(52 + (1-e*0.4)*35) : clamp(30 + e*38)
```

**崩壊判定**
```
crash = entropy > 76  ||  (infra < 18 && socialCap < 18)
```

**読み方**: フィルタリング率 `f` を上げるほど多様性・情報健全性が単調に低下し（`diversity = 100 - f*82`）、エントロピーが上がる。倫理 `e` が全体の健全度を底上げする。Greedy は各所にペナルティ `gp` を加える。

---

## L2 — 物理インフラ（`metricsP2(s, d, al, e)` / `calcRedundancyBuffer`）

**パラメータ**
- `sf = shrinkRate/100` … インフラ集約化率（スマートシュリンク）
- `df = dxRate/100` … 産業DX・法人化投資
- `ef = ethicsP2/100` … 行政リーダーの倫理観
- `isG` … 予算配分戦略が Greedy か
- `reboot = publicReboot` … 公共性リブート（再公営化）ON/OFF
- `dampener = reboot ? 0.5 : 1.0`

**冗長性バッファ（Redundancy Buffer）**
```
base = clamp(100 - sf*52 - (isG ? 32 : 0))
redundancy = reboot ? clamp(base + 42) : base
```

**インフラ・判定**
```
infraBase = clamp(100 - (1-sf)*35*dampener - (1-df)*15 - (isG ? (1-ef)*28 : 6)*dampener)
infraError = infraBase < 65

deadlock = (!rootRestricted) && ef < 0.4 && isG && infraError   // 他責デッドロック
infra    = deadlock ? clamp(infraBase - 42) : clamp(infraBase + (cpuRepair/100)*14)
heliOp   = reboot ? true : (infra > 35 && cpuRepair > 25 && ef >= 0.3)        // 救命ヘリ稼働（低倫理で停止）
budget   = clamp(100 - (1-sf)*44 - (isG ? (1-ef)*38 : 8) + df*14 - (reboot?16:0) - (rootRestricted?5:0))

crash = (!heliOp && infra < 20) || budget < 8 || <ショックでcrash>
```

**環境ショック注入**: 冗長性バッファが **< 30% で即崩壊**、**≥ 60% で生存**、30〜60% で部分損傷、という三分岐（`shockState`）。

**読み方**: 集約化 `sf` を極めるほど冗長性が失われ（`base = 100 - sf*52 - …`）、ショックに脆くなる。`deadlock` は「低倫理 × Greedy × インフラ不調」が揃うと発火し、CPUが100%他責化に回って修復が止まる。`heliOp` は倫理観 `ef >= 0.3` に直結し、低倫理では――インフラと修復予算が充分でも――ヘリが飛ばない（リーダーの判断力・協調性の欠如が実務を停止）。`reboot`（再公営化）は冗直性 +42・インフラ劣化半減で耐性を回復する。

## L3 — 個人の認知（ノードシミュレーション + `updateP3Monitor`）

**パラメータ**
- `depth = searchDepth`（1〜10）… 思考の探索深度
- `g = grounding/100` … 現実同期レート（SNS断食・一次情報）
- `lr = learningRate/100` … 学習率（自己修正の素直さ）。`stub = 1 - lr`（頑固さ）

**騙されやすさ・汚染の時間発展（エージェント）**
```
fool = depth<=3 ? 1 : depth>=7 ? 0 : (7-depth)/4     // 浅い探索ほど騙される
各ノードの汚染度 n.c += rise - fall
  rise = fool * 0.011 * (1 - g*0.75)
  fall = ( (1-fool)*0.012 + g*0.010 + <dropout> ) * lr   // ← 学習率0なら回復ゼロ
integrity = round(clamp(100 - avgC*100))              // avgC = 全ノード平均汚染
```

**回復タイムライン（予測）**
```
integ = round(clamp(95 - 78*fool*(1 - g*0.6) - stub*42))
```

**バナー判定（v6.338 以降・パラメータから即時判定）**
```
凍結(赤・アップデート拒否) : lr < 0.10
毒入れ(赤)                 : depth <= 3
健全(緑)                   : depth >= 7 && lr >= 0.30
注意(橙)                   : それ以外
```

**読み方**: `fall`（汚染の回復項）が **学習率 `lr` に比例**するため、`lr=0`（アップデート拒否）だと**どんな事実を入れても汚染が抜けない**（＝深く考えていても更新しない罠）。深い探索（`depth≥7`）＋更新可能（`lr≥0.3`）で初めて「健全」。

---

## L4 — 当事者性（`updateP4Monitor` / タイムライン）

**パラメータ**
- `gamification`（0〜100）… 目的関数のゲーム化（政策改善→敵味方ゲーム）
- `extTraffic`（0〜100）… 外部ノード流入率（Sybilフラッド）
- `jamRatio` … シミュレーション由来のリンク遮断率
- `filterActive` … PACKET_FILTERING 実行中か

**主要メトリクス**
```
drop  = round(clamp(jamRatio*72 + gamification*(filterActive ? 0.08 : 0.3)))  // 生活者パケットのドロップ率
ratio = round(clamp(100 - gamification*0.85 - (filterActive ? 0 : extTraffic*0.2)))  // 真の当事者比率
```

**バナー判定**
```
乗っ取り(赤)      : gamification > 70 && drop > 62   （extTraffic>60 なら「炎上の見世物」＝複合）
声が届く(緑)      : gamification <= 18 && extTraffic <= 18
注意(橙・観客に埋没) : それ以外
```

**読み方**: ゲーム化 `gamification` が支配的にドロップ率と当事者比率を悪化させる。フィルタリングは外部ノイズの寄与を落として当事者パケットを優先ルーティングに戻す。**批判対象は「よそ者」ではなく、当事者でない声（観客・偽装アカウント）が量で当事者の声を埋没させる非対称そのもの**。

---

## レイヤー間カスケード

- **L4 → L2**: `gamification > 70` でL2のインフラカードが赤く脈動（破壊的ストレス警告）。
- **L1 → L3**: `filterRate > 70` でL3の認知カードが紫〜赤に明滅（認知ハックの侵食）。
- **L3 → L2**: L3の「Root権限制限（多重承認）」が有効だと、L2の他責デッドロック `deadlock` の発火条件から外れる（`!rootRestricted`）。
- **L3 ↔ L2**: L3の「反証許容度」はL2の予算配分戦略（Greedy/DP）を参照する。

## GRAND OPTIMAL（全レイヤー健全）

各レイヤーのスナップショット（多様性・インフラ・認知健全性・当事者比率）が**すべて 70 以上かつクラッシュ無し**で認定。

---

## 前提・限界・免責（Assumptions & Limitations）

1. **係数は説明用の任意値**であり、実データからの推定ではない。数値の絶対値ではなく**変化の向き**を読むこと。
2. **単方向・即時反応**のモデルが中心（一部のみ時間発展）。実社会の遅延・フィードバックループ・非線形性は大幅に簡略化している。
3. **各メトリクスは独立式**で、相互作用は限定的（カスケードは明示した箇所のみ）。
4. **「良い/悪い」の割り当ては設計者の価値判断**を含む（例: 高倫理・低フィルタ・冗長性確保を「健全」とする）。これ自体を批判的に検討してよい。
5. **政治的対称性の担保**: 特定の党派・人物を指すものではなく、「どの社会にも起こる構造」を扱う。
6. **予測・診断・政策提言の根拠には使えない**。教育・思考実験・議論のきっかけとしての利用を想定。

> 数式やしきい値を変えたい・議論したい場合は、Issue（研究者・技術者向け）またはアプリ内フィードバックからどうぞ。**モデルを疑うことが、このアプリの正しい使い方です。**
