# タスク仕様書テンプレート（サブエージェント委任用）

> CLAUDE.md「委任プロトコル」で使う定型。メインセッションがこれを埋めて Agent tool（model: opus）に渡す。
> サブエージェントへの鉄則: **コミットしない・仕様外のファイルに触れない・不明点は推測せず報告して終了**。

```markdown
## タスク: <一行で>

### 目的（なぜやるか）
<戦略文書のどこに根拠があるか1〜2行>

### まず読むファイル（この順で）
1. CLAUDE.md（プロジェクト制約。特に integrity と i18n）
2. <対象に関係する既存ファイル・参考実装>

### 作成/変更してよいファイル（これ以外は触らない）
- <パスを列挙。新規/編集を明記>

### 要求仕様
- <箇条書きで具体的に。曖昧語を使わない>

### 禁止事項
- git commit / push（レビューは親が行う）
- 仕様外ファイルの変更・削除
- <タスク固有の禁止>

### 受け入れ条件（全部満たすまで完了と報告しない）
1. `npm run check` が green（prettier/eslint 含む）
2. <タスク固有の検証コマンドと期待結果>
3. （web/ に触れた場合）`npm run verify` が green

### 完了報告に含めるもの
- 変更ファイル一覧と各ファイルの要点
- 受け入れ条件の実行結果（コマンド出力の要約）
- 判断に迷った点・仕様の曖昧だった点
```
