# ユーザーインタビュー・ガイド（20人フェーズ / 15分 × 5人以上）

> PM.md「フェーズ1の20人は全員と話せる規模。5人以上に15分インタビューを行い、USの優先度をデータで見直す」の実施キット。
> 目的は褒め言葉の収集ではなく、**「何が刺さったか」と「どこで離脱したか」の2点の特定**。

## 原則（3つだけ）

1. **誘導しない。** 「面白かったですよね？」ではなく「最初の1分で何をしましたか？」。事実→感情の順で聞く。
2. **沈黙を埋めない。** 答えに詰まったら7秒待つ。次の質問に逃げると一番大事な部分が消える。
3. **その場で直すと約束しない。** 「持ち帰って検討します」でよい。ただし**1週間以内に反映 or お礼を必ず返す**（OUTREACH.md と同じ作法。ここまでがインタビュー）。

## 冒頭の断り（30秒・毎回同じ）

```
15分だけ、使ってみた感想を聞かせてください。開発者に気を使う必要はまったくなくて、
「分からなかった・つまらなかった」が一番役に立つ情報です。
記録はこちらのメモだけで、名前は出しません。
```

## 質問スクリプト（15分）

### A. 再現（5分）— 何をしたかの事実

1. アプリを開いて、**最初の60秒**で何をしましたか？（画面を見せてもらえるならその場で再現してもらう）
2. 最初に「お」と思った瞬間はどこですか？ 逆に「？」となった瞬間は？
3. どこで閉じましたか？（離脱点。ここが最重要。「全部見た」なら何周目で飽きたか）

### B. 理解（5分）— 何が伝わったか

4. このアプリを友だちに一言で説明するとしたら、何と言いますか？（＝伝わったコンセプトの実測）
5. 4ページのうち、どれが一番刺さりましたか？ どれが要らないと思いましたか？
6. （PAGE 2 を見た人に）「効率を上げたのに災害に弱くなる」は体験の中で腹落ちしましたか？

### C. 行動（5分）— 次につながるか

7. 誰かに共有しようと思いましたか？ **しなかったなら、何がブレーキでしたか？**（政治的に見られる不安か・面白さ不足か・操作か）
8. 1週間後にもう一度開くとしたら、何があれば開きますか？（週替わりシナリオの需要検証）
9. （研究者・教員のみ）これを授業・研究で使うとしたら、**何が足りませんか？**

## 記録テンプレ（1人1行で `docs/interview-log.md` に追記）

| 日付 | 属性(一般/研究者/教員) | 最初の60秒 | 刺さった点 | 離脱点/？の瞬間 | 一言説明(Q4の実言) | 共有ブレーキ | 反映したUS/Issue | 報告済 |
|---|---|---|---|---|---|---|---|---|
|  |  |  |  |  |  |  |  |  |

- **Q4 の「一言説明」は原文のまま**書き取る（これが次の共有テンプレ・ストア説明文の素材になる）。
- 5人終わったら PM.md のユーザーストーリー優先度を見直し、変更点を CLAUDE.md 進捗ログに1行残す。

## 分析の型（5人分たまったら30分）

1. 離脱点を並べる → 2人以上が同じ場所で落ちていたら、それが最優先の修正。
2. Q4 の一言説明を並べる → 自分の想定コンセプト（「体験できる構造の話」）とズレていたら、刺さった方に寄せる。
3. 共有ブレーキの内訳 → 「政治的に見られる不安」が多ければ免責・作法カードの露出を強化、「面白さ」なら週替わり・発見ログを強化。
