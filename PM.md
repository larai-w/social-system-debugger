## 1. プロダクトビジョン

社会は壊れやすい。しかし一人ひとりが構造を理解しアクションを起こすことで未来は変えられる。
それを説明ではなく「操作できる体験」として届ける。
- Web = 60秒のお試し（URLひとつ、ログインなし、プリセット1タップで崩壊を見る）
- モバイル = 本格的な学びと共有と行動（週替わりシナリオ、蓄積、現実へのアクション）

**North Star Metric: 「守られた街」の累積数**（週次シナリオクリア＋巻き戻し成功の合計）。
補助指標: 週次リテンション（weekly_start のユニーク推移）、共有実行数（share_x / share_line）、
現実アクションのチェック数（フェーズ3〜）。

## 3. ユーザーストーリー

### フェーズ1（〜20人）: モバイル基盤と「帰ってくる理由」

- ✅ **US-01** (P0) 初見ユーザーとして、URLを開いて60秒以内にプリセット1タップで社会が壊れる様子を見たい。説明を読まされたくない。
  - 受入条件: 初回モーダルはスキップ可 / ヘッダー要素は5つ以下 / プリセットがファーストビューにある
- ✅ **US-02** (P0) ユーザーとして、街に名前を付けて「自分の街」の物語として体験したい。
  - 受入条件: 1入力のみ・スキップ可・`ssd_town_name` 永続化・バナーと結果カードに反映
- ✅ **US-03** (P0) ユーザーとして、崩壊後に「1つだけ過去に戻して変えられるなら」を試して、自分の介入で未来が変わることを体験したい。
  - 受入条件: PAGE 1で巻き戻し→再実行→生存時に結果カードへ一文追加
- **US-04** (P0) モバイルユーザーとして、アプリをストアからインストールして、Webと同じ体験＋週替わりシナリオを使いたい。
  - 受入条件: Capacitorビルドが iOS/Android で起動 / `WEEKLY_ENABLED` がネイティブで自動ON
- **US-05** (P0) アプリユーザーとして、週1回の通知で新しいシナリオが来たことを知り、3分で挑戦したい。
  - 受入条件: ローカル通知（曜日・時刻固定）/ 通知許可は初クリア直後に文脈付きで要求 / 挑戦→判定→結果カードまで3分以内
- **US-06** (P0) ユーザーとして、結果カードをX・LINE・画像保存の3経路で、自分の一言を添えて共有したい。
  - 受入条件: 3ボタン明示 / 一言入力(スキップ可) / track イベント share_x・share_line・card_saved が飛ぶ
- **US-07** (P0) 開発者として、シナリオJSONを1ファイル書いてPRを出すだけで、検証→S3反映→latest.json更新まで自動化したい。
  - 受入条件: GitHub Actions（OIDC・長期キーなし）/ スキーマ検証で ja/en 欠落を検出 / check関数は宣言的条件配列に移行済み
- ✅ **US-08** (P1) 研究者として、現在のパラメータと結果をJSON/CSVでエクスポートして教材や資料に使いたい。
  - 受入条件: 全パラメータ＋主要メトリクスのエクスポート / 再現用の共有URLと同内容
  - **この機能が生む価値（立場別）**:
    - *一般ユーザー・生徒*: 「遊んだ結果」が手元に残るデータになる。CSVはスプレッドシートに直接貼れ、設定を変えた実験の比較や自由研究・レポートの素材になる。JSONには再現用共有URLが入り、「あの時の街」を1クリックで復元できる（classroom.html の探究の型はこれが前提）。
    - *研究者*: 最大の価値は**再現性**。結果をデータ＋再現URLで受け渡しでき、全メトリクスの数値から感度分析（倫理を10下げると正当性がいくつ落ちるか等）を外部ツールで検証できる。OUTREACH の「モデルの粗を1つ指摘してほしい」に、具体的な数字で答えてもらえるようになる。
    - *開発者*: ①フィードバックの質が上がる（「なんか変」→「このJSONの状態で brand がこの値なのはおかしい」）。②就活・出願で「研究利用を想定してエクスポートと再現性を設計した」と語れる＝CSの研究文化理解の証明。③ユーザーインタビュー時に相手の実験データを見ながら話せる。

### フェーズ2（〜50人）: みんなで守ってる感

- **US-09** (P0) ユーザーとして、アプリを開いたら「これまでに全国で N の街が守られました」という累積を見て、自分が集合的な営みの一部だと感じたい。
  - 受入条件: 共同カウンターAPI（API Gateway+Lambda+DynamoDB）/ オフライン時はローカル値でフォールバック
- **US-10** (P0) ユーザーとして、週替わりシナリオをクリアしたら「今週、街を守った N 人のひとり」として街名と一言を署名に残したい。
  - 受入条件: 署名は任意 / 表示は街名＋一言のみ（個人特定情報なし）/ 不適切語フィルタ
- **US-11** (P1) ユーザーとして、過去の週のシナリオアーカイブに挑戦して、クリア履歴を図鑑に貯めたい。
- **US-12** (P1) 教員として、授業で使うために、生徒に同じシナリオを一斉に配りたい（簡易: シナリオ固定URLで代替可）。

### フェーズ3（〜100人）: 現実への接続

- **US-13** (P0) ユーザーとして、シミュレーション後に「自分の現実の街でできる一歩」（ハザードマップ確認等）を提示され、実行したらチェックして記録したい。
  - 受入条件: 診断は架空・行動は現実の分離を守る / チェックは自己申告と明記 / 共同カウンターに「現実のアクション」として合算
- **US-14** (P1) ユーザーとして、SWAP THE LEADER 実験で「トリガーは原因ではない、攻撃面が原因」という診断を体験し、カードとして共有したい。
  - 前提: design-note-page5.md の発動条件を満たすこと
- **US-15** (P2) 研究者・上級ユーザーとして、PAGE 5 比較OS診断で2つの社会OSに同一ショックを注入して比較したい。
